La vue Supports

Cette vue affiche les supports que vous avez créés et permet de les gérer. Veuillez lire la page Utilisation des Supports pour connaître les principes des supports.

Notez que vous pouvez également crééer des supports de type Répertoire en utilisant l'assistant support (Menu général -> Aide -> Assistant Supports).

Le référentiel des supports

Dans la partie basse de la vue Support, vous trouvez les supports que vous avez créés. La flèche verte en bas du support signifie que le support est monté.

 Ceci est simplement un raccourci pour créer un nouveau support.

 CD de données.

 Répertoire en local.

 Lecteur réseau (Netbios/Samba/NFS...). Sous Windows, utilisez un chemin par lettre (comme T:\...) et non par favoris réseau ou chemin UNC.

 Baladeur MP3.

 Disque externe ou support de stockage comme disque externe USB/Firewire ou clé USB.

 Noeud Jajuk distant (pas encore implémenté)

Le menu contextuel sur les supports sont les suivants (veuillez lire la page Utiliser les supports )

  • Monter: Monte le support

  • Démonter: Démonte le support.

  • Test: Teste si le support est accessible.

  • Rafraîchir: lance le rafraîchissement du support.

  • Synchroniser (disponible si ce support est configuré pour se synchroniser à un autre support) : lance la synchronisation.

  • Supprimer le support: Supprime le support de la collection (aucune donnée ne sera effacée du disque). Vous pouvez configurer l'interface de confirmation dans la vue Paramétrage / onglet Confirmations.

  • Configuration: Gère le paramétrage du support.

Le panneau de configuration

Le panneau de configuration dans la partie haute de la vue propose de :

 Créer un nouveau support (vous pouvez aussi cliquer sur l'icône "Nouveau").

 Supprimer un support de la collection (aucune donnée sera effacée du disque). Vous pouvez configurer l'interface de confirmation dans la vue Paramétrage / onglet Confirmations.

 Configurer le support sélectionné.

 Monter le support sélectionné.

 Démonter le support sélectionné.

 Tester si le support sélectionné est accessible.

 Supprimer le support sélectionné.

 Synchroniser le support sélectionné (disponible si ce support est configuré pour se synchroniser à un autre support).

Panneau de configuration

Si vous ne connaissez pas les notions Jajuk de montage, rafraîchissement ou synchronisation, veuillez consulter la page Utiliser les supports.

Cette fenêtre apparaît lorsque vous créez ou modifiez la configuration d'un support existant. Vous pouvez afficher la configuration d'un support en cliquant dessus ou en utilisant le menu contextuel / configuration. La fenêtre propose les options suivantes:

  • Type de support (non modifiable): choisissez le type de support à créez, voir les types plus haut.

  • Nom du support (non modifiable): saisissez le nom du support comme "CD A" pour un CD de données ou "Portable" pour un répertoire du disque dur.

  • Chemin du support (non modifiable): chemin complet du support, comme "~/musique" sous Unix ou "d:\musique" sous Windows. vous pouvez utiliser le sélecteur de répertoire sur la droite.

  • Point de montage Unix (seulement sous Linux/Unix). Cette option est optionnelle et permet à Jajuk de tenter de monter un système de fichier par la commande "mount" si besoin lors du montage Jajuk d'un support. Par exemple, si vous désirez monter un support de type CD dont le point de montage est /mnt/cdrom et que le CD n'est pas monté, Jajuk le montera pour vous. Notez cependant que cette option est inutile sur la plupart des distributions Linux récentes qui propose un auto-montage.

  • "Faire un rafraîchissement immédiat" (défaut = vrai): effectue un rafraîchissement immédiat après la création du support (recommandé). Notez que cela peut prendre un temps important, surtout lors du premier rafraîchissement.

  • "Montage auto au démarrage" (vrai par défaut pour certains types de supports et à faux pour d'autres): Effectue un montage automatique au lancement de la session Jajuk. C'est recommandé pour les répertoires et les supports fixes. A éviter pour les supports externes comme les CD ou les clés USB.

  • "Rafraîchissement auto au démarrage (faux par défaut)": rafraîchit le support à chaque démarrage de Jajuk: déconseillé, surtout pour les gros supports.

  • Synchronisation: si vous disposez de plus d'un support, cette boite permet de les synchroniser. Le support courant sera alors le support cible et l'autre le support source.

    • "Synchronisation unidirectionnelle": tous les nouveaux morceaux du support source sont copiés vers ce support.

    • "Synchronisation bidirectionnelle": tous les nouveaux morceaux d'un support sont copiés sur l'autre.

    Notes:

    • Vous pouvez ignorer certains répertoires lors de la synchronisation (si les deux supports n'ont pas la même capacité disque par exemple) en désynchronisant les répertoires dans l'arbre Physique/ menu contextuel sur un répertoire.

    • La synchronisation n'efface jamais de fichiers donc si vous renommez des fichiers sur le support source, vous obtiendrez deux versions des fichiers: l'ancienne et la nouvelle.
