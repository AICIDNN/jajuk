Utiliser des supports

Définition

Jajuk joue de la musique à partir de supports. Un support est un ensemble de fichiers audio. Cela peut être :

  • Un répertoire

  • Un CD-ROM de données

  • Un disque dur externe

  • Un lecteur réseau (Netbios/Samba/NFS...). Sous Windows, utilisez des chemins de lecteurs réseau (comme T:\...) et non des favoris réseau ou des chemins UNC.

  • Un lecteur physique tel qu'un iPod

  • Un répertoire distant (non implémenté)

Note: Dans tous les cas, excepté les répertoires distants, les supports sont des répertoires standards du point de vue du système mais Jajuk différencie les cas pour traiter les particularités (par exemple, un CD-ROM est en lecture seule) et ainsi fournir une interface graphique adaptée.

Un support contient généralement des fichiers MP3, OGG et d'autres formats audio supportés, des fichiers pour playlistes (ou liste de lecture) ainsi que des JPG, GIF et autres formats images utilisés par les jaquettes.

Ce que nous nommons collection est une combinaison de supports que vous avez créés.

Rafraîchir les supports

Sous Jajuk, les supports peuvent être rafraîchis de temps à autre. En effet, ce qui rend Jajuk si rapide est son indexation des collections : les supports de musique ne sont pas scannés et leurs meta-données ne sont pas lus à chaque démarrage mais Jajuk construit une collection ou un index durant le rafraîchissement (enregistré sur le disque dans {répertoire personnel}/.jajuk/collection.xml). Cela permet un démarrage rapide et la connaissance en quelques secondes de tous les fichiers audio sur plusieurs centaines de Go de collection musicale. Cela permet aussi de filtrer et de réaliser de puissantes recherches sur une collection complète en temps réel.

Jajuk ne reconnaît les nouveaux fichiers que lors d'un rafraîchissement des supports: si vous copiez un nouveau fichier sur votre disque, il ne sera pas connu tant que vous n'aurez pas un rafraîchi le support source.

Les supports peuvent être rafraîchis à chaque démarrage mais ce n'est pas recommandé pour les supports volumineux car cela prend beaucoup de temps. Notez cependant que Jajuk intègre des optimisations qui accélère grandement le rafraîchissement: par défaut, Jajuk ne lit pas les meta-données des fichiers connus. Vous pouvez le forcer à les lire avec l'option "Forcer la lecture des meta-données" dans la vue Paramètre/Onglet Option si vous changez les meta-données en utilisant une autre application.

Vous pouvez rafraîchir un support depuis la "Perspective Physique" avec la commande "Rafraîchir" depuis le menu contextuel du support sélectionné ou depuis la vue Supports dans la perspective Configuration.

Monter/Démonter des supports

Monter un support signifie le rendre disponible (donc connu de Jajuk) et le démonter signifie le déconnecter. C'est l'équivalent des notions de "Mount" et "Unmount" que l'on trouve sous Unix. Vous pouvez configurer les supports dans la vue Support afin qu'ils soient automatiquement montés au démarrage. Certains supports peuvent naturellement être auto-montés comme par exemple un répertoire depuis votre disque dur interne. D'un autre côté, nous décourageons l'auto-montage des supports sources qui ne sont pas "stables", c'est-à-dire les supports CD ou les clés USB (par exemple), car il est probable qu'ils ne soient plus disponibles au prochain démarrage de Jajuk. Quand un support requiert un montage, Jajuk va vérifier la disponibilité physique de ce dernier (par exemple la présence du CD dans le lecteur) et va essayer d'y accéder sur demande. Sous Unix, il se peut que cela exécute une commande "Mount". Si vous essayez de jouer un morceau situé sur un support non monté, alors Jajuk vous demandera si vous voulez le monter. Dans tous les cas, vous aurez un message d'erreur si le support ne peut être monté

Vous pouvez monter ou démonter un support depuis la perspective physique avec les commandes "monter ou démonter" du menu contextuel du support sélectionné ou depuis la vue Support dans la perspective Configuration.

Synchroniser les supports

Quand on crée un support (voir la documentation sur la vue périphérique pour plus d'informations), vous pouvez demander à Jajuk de le synchroniser avec un autre. Cela vous permet de copier en un simple clic tous les nouveaux fichiers d'un support source à un autre. Notez que vous pouvez effectuer une synchronisation partielle par la désynchronisation de quelques répertoires dans la perspective Physique option Arbre Physique ou par le menu contextuel sur le répertoire choisi (voir la boîte de sélection en bas de la vue Information pour connaître la taille récursivement de la sélection). Vous pouvez aussi effectuer des synchronisations bi-directionnelles (tous les nouveaux fichiers d'un support sont copiés vers l'autre) et pas seulement unidirectionnelles.

Vous pouvez synchroniser deux supports depuis la perspective physique avec la commande "Synchroniser" du menu contextuel du support sélectionné ou depuis La vue Support dans la perspective Configuration.

Sauvegarde de la collection

Toutes vos collections et les propriétés des morceaux sont enregistrées dans un fichier collection (fichier {répertoire utilisateur}/.jajuk/collection.xml). Par défaut Jajuk effectue une sauvegarde de ce fichier (collection-<date>.xml files) jusqu'à ce que sa taille atteigne les 20 MB sur le disque.

Vous pouvez paramétrer la sauvegarde dans la vue Paramètre option Options Avancées.
