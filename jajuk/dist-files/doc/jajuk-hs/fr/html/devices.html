Utiliser des supports

Définition

Jajuk joue de la musique à partir de supports. Un support est un ensemble de fichiers audio. Cela peut être :

  • Un répertoire

  • Un CD-ROM de données

  • Un disque dur externe

  • Un lecteur réseau (Netbios/Samba/NFS...). Sous Windows, utilisez des chemins de lecteurs réseau (comme T:\...) et non des favoris réseau ou des chemins UNC.

  • Un baladeur MP3.

  • Un répertoire distant (non implémenté)

Note: Dans tous les cas, excepté les répertoires distants, les supports sont des répertoires standards du point de vue du système mais Jajuk les différencie pour traiter les particularités (par exemple, un CD-ROM est en lecture seule) et ainsi fournir une interface graphique adaptée.

Un support contient généralement des fichiers MP3, OGG et d'autres formats audio supportés, des fichiers pour playlistes (ou liste de lecture) ainsi que des JPG, GIF et autres formats images utilisés par les pochettes.

Votre collection contient tous les supports que vous avez créés.

Créer et configurer les supports

Les supports peuvent être créés soit via la vue Supports dans la perspective Configuration soit en utilisant l'assistant support (Menu général -> Aide -> Assistant Supports).

Rafraîchir les supports

Jajuk utilise un système d'indexation automatique: les supports de musique ne sont pas scannés et leurs meta-données ne sont pas lues à chaque démarrage mais Jajuk construit une collection durant le rafraîchissement (enregistré sur le disque dans {répertoire personnel}/.jajuk/collection.xml). Cela permet un démarrage rapide et la connaissance en quelques secondes de tous les fichiers audio sur plusieurs centaines de Go de collection musicale. Cela permet aussi de filtrer et de réaliser de puissantes recherches sur une collection complète en temps réel.

Jajuk ne reconnaît les nouveaux fichiers qu'après le rafraîchissement du support: si vous copiez un fichier sur votre disque, Jajuk en sera informé qu'après le rafraîchissement suivant. Le rafraîchissement est automatique et sa fréquence est paramétrable dans la vue Supports. La fréquence par défaut pour un disque dur (Support de type Répertoire) est de 30 secs (0.5 mins) et le scan dure en général seulement quelques secondes donc si vous supprimez ou ajoutez des fichiers dans ce support, le changement sera reflété dans l'interface graphique après environ 30 secs. Notez que la fréquence pour les lecteurs réseau est beaucoup plus importante (défaut: 5 mins) car le scan dure plus longtemps (1 min ou plus pour les gros lecteurs) et consomme de la bande passante.

Jajuk ne relit pas les tags des fichiers connus. Vous pouvez le forcer à les lire avec la fonction "Forcer le rafraîchissement" dans l'arbre physique sur les supports ou dans la vue Supports si vous changez les tags en utilisant une autre application.

Monter/Démonter des supports

Monter un support signifie le rendre disponible (donc connu de Jajuk) et le démonter signifie le déconnecter. Vous pouvez configurer les supports dans la vue Support afin qu'ils soient automatiquement montés au démarrage. Certains supports peuvent naturellement être auto-montés comme par exemple un répertoire depuis votre disque dur interne. D'un autre côté, nous décourageons l'auto-montage des supports sources qui ne sont pas "stables", c'est-à-dire les supports CD ou les clés USB (par exemple), car il est probable qu'ils ne soient plus disponibles au prochain démarrage de Jajuk. Quand un support requiert un montage, Jajuk va vérifier la disponibilité physique de ce dernier (par exemple la présence du CD dans le lecteur) et va essayer d'y accéder sur demande. Sous Unix, il se peut que cela exécute une commande "Mount". Si vous essayez de jouer un morceau situé sur un support non monté, alors Jajuk vous demandera si vous voulez le monter. Dans tous les cas, vous aurez un message d'erreur si le support ne peut être monté

Vous pouvez monter ou démonter un support depuis la perspective physique avec les commandes "monter ou démonter" du menu contextuel du support sélectionné ou depuis la vue Support dans la perspective Configuration.

Synchroniser les supports

Lorsque on créé un support (voir la documentation sur la vue périphérique pour plus d'informations), il est possible de demander à Jajuk de le synchroniser avec un autre. Cela vous permet de copier en un simple clic tous les nouveaux fichiers d'un support source à un autre. Notez que vous pouvez effectuer une synchronisation partielle en désynchronisant des répertoires dans la perspective Physique option Arbre Physique par le menu contextuel sur le répertoire choisi (voir la boîte de sélection en bas de la vue Information pour connaître la taille récursivement de la sélection). Vous pouvez aussi effectuer des synchronisations bidirectionnelles (tous les nouveaux fichiers d'un support sont copiés vers l'autre) et pas seulement unidirectionnelles.

Vous pouvez synchroniser deux supports depuis la perspective physique avec la commande "Synchroniser" du menu contextuel du support sélectionné ou depuis La vue Support dans la perspective Configuration.

Sauvegardes et restaurations

Toutes les données de votre collection sont stockées dans le fichier collection ({répertoire personnel}/.jajuk/collection.xml). Par défaut, jajuk créé des sauvegardes de cette collection (fichiers collection-{date}.xml) jusqu'à occuper 20Mo sur le disque. Vous pouvez paramétrer la sauvegarde dans l'onglet Avancé de la vue Paramétrage. 

Restauration: Si vous faite une erreur de manipulation et que vous êtes toujours dans jajuk, fermez l'application puis écrasez le fichier collection_exit.xml avec le fichier collection.xml. Si vous voulez revenir à une version plus ancienne, écrasez le fichier collection_exit.xml avec le fichier collection-{date}.xml file. Redémarrez ensuite jajuk.

Enfin, il serait bon d'inclure votre répertoire {répertoire personnel}/.jajuk dans vos sauvegardes régulières.
