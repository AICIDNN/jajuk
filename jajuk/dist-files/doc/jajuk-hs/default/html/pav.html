Perspectives and views

Definitions

  • Jajuk is divided into perspectives and views. A perspective is a logical set of views. For instance, you can display the physical perspective to navigate into disk files, directories or the logical perspective to navigate into genres, artists, albums and tracks.

  • A view is located inside a perspective and shows some linked information. For instance, the Physical perspective contains both the Physical Tree view and the Physical Table view.

  • To change of current perspective, click on the perspective icon of your choice in the Perspective Selector view (see figure 02).

Jajuk perspectives

Jajuk release contains following perspectives :

 Physical perspective: in this perspective, your collection is shown as it is on your disks: files, directories, m3u files etc. All actions are performed on an given physical file. This perspective is useful but note that if you have several times the same track in different devices (lets say one on your hard disk and one on a CD), you will see it several times in all views. This is the same for playlist files.

 Logical perspective: in this perspective, your collection is shown following id3 tags information like genres, artists, albums, tracks and logical playlists (a logical playlist maps several identical physical m3u playlists). This perspective is useful only if your collection is correctly tagged, otherwise everything will be 'unknown'. In opposition to the physical perspective, you now see tracks, not files. A track is a logical mapping to one or more files. It means you will see a track only once even if you have several physical files for this track. If you modify a logical playlist, all associated physical playlists (.m3u files) will be modified together.

 Player perspective: used to show various information while playing tracks: one animation view with large text area to display track names and two cover views.

  Catalog perspective: display collection using interactive covers thumbnails.

 Configuration perspective: this contains views to configure Jajuk (options) and to create or modify devices. A device is a set of physical files (like a directory on your hard disk, a CD, a music player...).

 Statistics perspective: contains various pies and histograms about your collection: number of tracks, devices sizes, genres repartition, collection size.

 Help perspective: Jajuk help and 'About' view providing various information on the project and the runtime environment.

Using docking

Jajuk interface can be organized just like you want it: you can resize, minimize, drag a view and add it to another view as a tab, make view auto-hideable (using the drawing-pin)... 

You can reset default views configuration by using the top command view: Views -> restore defaults. 

Views configuration are saved when closing Jajuk.
