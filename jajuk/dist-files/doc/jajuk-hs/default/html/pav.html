Perspectives and views

Definitions

  • Jajuk is divided into perspectives and views. A perspective is a logical set of views. For instance, you are currently in the Help perspective to get some help and you will use physical perspective to navigate into disk files, directories or  logical perspective to navigate into styles, authors, albums and tracks.
  • A view is a way to show informations relative to a perspective. For instance, you have both physical tree view and the physical table view in the physical perspectives.
  • There is two types of views : static ones ( command, information and perspective bar ) which are present in all perspectives and contextual ones which are related to a perspective. Note that a same view ( even regular ) can be found in several perspectives ( cover view for instance ).
  • To change current perspective, click on perspective icon of you choice in the perspective bar view ( see figure 02 ).

Jajuk perspectives

Jajuk release contains following perspectives :

 Physical perspective: In this perspective, you see your collection as it is on your disks : files, directories, m3u files etc...  All actions are performed on an identified physical file. This perspective is useful but has a big disadvantage : if you have several times the same track in different devices ( lets say one on your hard disk and one on a CD ), you will see it twice in all views. This is the same for playlists.

 Logical perspective: In this perspective, you see your collection following id3 tags informations : as styles, authors, albums, tracks and logical playlists ( a logical playlist maps several identical physical m3u playlists). This perspective is  useful only if your collection is well tagged, otherwise everything will be ' unknown'. In opposition to physical playlist, you now see tracks, not files. A track is a logical mapping to one or more files. It means you will see a track only once even if you have several physical files for this track. If you modify a logical playlist, all associated physical playlists ( .m3u files ) will be modified together.

 Player perspective: Used to show static information while playing tracks : one animation view with large text area to display track name and two cover views.

 Configuration perspective: This contains views to configure Jajuk ( options ) and to create or modify devices. A device is a set of physical files ( like a directory on your hard disk, a CD, a music player ...).

 Statistics perspective: Contains various pies and histograms about your collection : number of tracks, devices sizes, style repartition, collection size. More stats will be available in next releases.

 Help perspective: Should be this current perspective. Contains Jajuk help and a 'About' view giving various informations on the project and the runtime environment.

Using docking

Jajuk interface can be organize just like you want: you can resize, minimize, drag a view and add it to another view as a tab... If you are in trouble, you can reset default views configuration by using the top command view : Views-> restore defaults. Of course, your views positions and configurations are saved when closing Jajuk.
