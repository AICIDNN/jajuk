Performance tips

Read that chapter if you encounter some performances drawbacks or if you want consume less memory, disk, CPU or Network bandwidth using Jajuk.

JVM choice and customization

We recommend Sun JVM 1.5.0 that is from 10 to 30% faster that the 1.4.2 running Jajuk. Let us know if you manage to run Jajuk with another JVM, especially free ones (Kaffe, SableVM...).

Default command line to launch Jajuk is :

java -client -Xms25M -Djava.library.path=../native -jar jajuk.jar -notaskbar

  • If you have a really big collection (something like > 200GB), you may increase -Xms parameter.

  • You can set a -Xmx parameter to restrict memory usage to this value (example: -Xmx100M), but this could conduct to unpredictable behavior when out of memory errors occur.

  • Use -Xloggc:<file name> to trace memory usage if needed.

CPU usage

  • On a PC/P4M 1.8Ghz, Jajuk uses about 9% CPU for mp3 and 3% for Ogg Vorbis.

  • Performances are better (by 50%) with Ogg Vorbis than MP3.

Memory Usage

A JVM runs inside a reserved memory heap. The JVM takes all the Xms MB of memory at startup and requires more memory to the OS only if necessary. For a big size collection and all cover options enabled, it takes about 65MB maximum of memory but 40MB should be enough most of the time.

Covers performance tunning

These options are available in the Parameter View / Cover tab.

  • “Adapt Cover” option (default=no) uses a lot of CPU and memory to generate and display a new image of the given size. Avoid this option if you have low CPU or Memory computer.

  • “Use auto-cover” (default=yes) uses network bandwidth, especially if you use “Preload Cover” option. Avoid this option if you have a low bandwidth.

  • “Preload covers” (default=no) is very network bandwidth consuming. Use this option only if you have a fast network like xDSL or cable.

  • You can reduce covers size by setting lower minimum and maximum cover sizes.

  • You can increase or decrease the time out when downloading a cover in the Parameter view/ Network tab.

Tags performance tunning

These options are available in the Parameter View / Tags tab.

  • Do not use “Perform a deep tag scan” (default=no) option unless you want to force tags read. By default, Jajuk doesn't read id3 tags at each refresh for known files.

Collection performance tunning

These options are available in the Parameter View / Advanced tab.

  • If your collection uses mainly non-European characters (like Asian languages), choose UTF-16 as collection encoding : that will reduce collection file size and accelerate startup parsing. Do not use this option if you use European languages because you collection file would then grow.

  • You can set the backup size for your collection. The bigger it is, the more you get backup files (one by day).
