The Physical Tree view

This view can be found in the Physical perspective. It allows you selecting or browsing your devices upon a physical point of view. In this view, you will find:

  • Devices as you created them in the Device view. Devices contain directories.

  • Directories

  • Audio files that Jajuk supports

  • Playlist files

Notes:

  • Cover files are not visible in the tree.

  • Tree configuration (nodes expands/collapses) is saved for next Jajuk session.

You can select any item in the tree, all contextual menu action will be applied recursively. For example, if you selected both your "ROCK" main directory and your "POP" main directory, then you right click and select "Play shuffle", Jajuk will play randomly all your ROCK and POP tracks.

Dragging and dropping items

At any time, you can drag a selection and drop it over the playlist you want to add selection to (in Playlist Repository view) or over the Playlist Editor View of current Playlist. If current playlist is the queue Playlist (default), a drag on the selection and drop over the Playlist Editor View will result into an immediate play of your selection.

Devices contextual menu

Device contextual menu features:

  • Play: Stop current track and play selection

  • Push: Don't stop current track but simply add the selection into the queue (see Queue playlist)

  • Play shuffle: Stop current track and play selection randomly (note that this mode is automatically used when playing a track if you set Shuffle mode in command view)

  • Play repeat: Stop current track and play selection in a loop (note that this mode is automatically used when playing a track if you set Repeat mode in command view)

  • Mount: perform a mount (see Using devices)

  • Unmount: perform an unmount (see Using devices)

  • Refresh: Perform a device refresh (see Using devices)

  • Synchronize: Perform a synchronization (see Using devices). It is enabled only if the device is configured to be synchronized with another one.

  • Test: check (or "ping") the device to check if it is really available.

  • Create Playlists (not yet implemented). Create playlists files.

  • Set a property (not yet implemented). Set a property to this item

  • Properties (not yet implemented). Show item properties.
