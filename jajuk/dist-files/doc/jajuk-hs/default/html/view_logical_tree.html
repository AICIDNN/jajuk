The Logical Tree view

This view can be found in the Logical perspective. It allows you to select or browse your devices upon a logical (style/author/album/track) point of view. In this view, you will find :

  • Styles (like Rock, POP, Classic...).
  • Authors
  • Albums
  • Tracks (note that a single track can map several identical files )
Notes :
  • Cover and playlist files are not visible in the tree.
  • Tree configuration (nodes expands/collapses) is saved for next Jajuk session.

You can select any item in the tree, all contextual menu action will be applied recursively. For exemple, if you selected both "Acoustic" and "Blues" styles, then you right click and select "Play shuffle", Jajuk will play randomly all your Acoustic and Blues tracks.

Draging and dropping items

At any time, you can drag a selection and drop it over the playlist you want to add selection to (in Playlist Repository view) or over the Playlist Editor View of current Playlist. If current playlist is the queue Playlist (default), a drag on the selection and drop over the Playlist Editor View will result into an immediately play of your selection.

Styles contextual menu

Styles contextual menu features:

  • Play: Stop current track and play selection
  • Push: Don't stop current track but simply add the selection in the queue (see Queue playlist)
  • Play shuffle: Stop current track and play selection randomly (note that this mode is automaticaly taken when playing a track if you used Shuffle mode in command view)
  • Play repeat: Stop current track and play selection in loop (note that this mode is automaticaly taken when playing a track if you used Repeat mode in command view)
  • Delete(not yet implemented): Delete selection
  • Set a property (not yet implemented). Set a property to this item
  • Properties (not yet implemented). Show item properties.
