Scenario-based tests

White: not tested, green: passed, orange: passed partially, red: not passed

#
 Description
 Result env1
 Result env2 Result env3
PKG
 Packaging tests : install Jajuk in several languages. Target directory : /prog/jajuk and c:\Program files\jajuk. Version overwriting tests : install Jajuk on existing release directory, then  remove the directory and install it to this unexisting location. Start and close Jajuk at each test.


SRT
 Tests with existing collection : start jajuk, lauch some tracks.


SCR
 Tests from scratch. Remove .jajuk directory, create a directory device with auto-refresh, lauch some tracks and then quit Jajuk and start it again, lauch some tracks, quit and test again.


FOR
 Tests on formats : lauch at least one mp3, ogg, au, aiff and wav file.



DEV
 Devices tests : create a directory device, refresh it, unmount it, mount it, remove it.


SYN
 Device synchronization : test with a directory device and a player device, if possible, try to mix FS : ext3 for directory, FAT32 for player.


SRC
 Instant search box : perform some searchs and lauch tracks.


HST
 History : check all lauched tracks are OK. Clear hisstory and lauch others tracks, check it again.


COP Compile and test the installed jajuk sources


SUP
 Startup tests : change startup option and test it.


BUP
 Backup tests : change backup options and test them.


LNG
 Test Jajuk en english and at least with 2 others langpacks, check translation for untranslated fields if any.


LNF
 Change Look and Feel, test it.


STA
 Check stats


TIM
 Check elapsed / remaining time in information bar


DOC
 Check About and online doc in all languages.


COM
 Command panel tests : rewind, forward, volume...


PLA
 Physical and logical playlists tests.


COV
 Tests covers display and manipulation


VIE
 Change views size / position, close jajuk and restart, check position is saved. Reload default perspectives in Parameter view / Perspective tab, test.


RES
 Change resolution (1024x768 to 1600x1400) and check everyhting's OK


ERR
 Rename or delete physically some files in your collection without refreshing device to simulate read errors, check behavior (error message in information panel, history bar...)


TRA
 Tray tests


RAN Random tests, general use tests in every views.
