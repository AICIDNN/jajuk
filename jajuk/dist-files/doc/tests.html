C_Synchro
 Test basic and advanced synchonization features. Start to create at least 2 devices whose one is synchronized with the other.
* Test full unidirectional sync. : make sure no directory is desynchronized in the physical tree view, set Unidirectional as device property, launch sychronization, check output and generated files.
* Test partial unidirectional sync. : same than previus but desycnrhonize some directories in the physical tree view, lauch sync. ,check output and generated files and especially that desycnrhonized directories are not sycnrhonized.
* Test full bidirectional sync.
* Test partial bidirectional sync.

 Devices are correctly sychronized :
- Both devices are refreshed before sync.
- In unidirectional sync, all new files in the source device must be copied to the current device.
- In bidirectional sync, both device will contain the same files after sync.
- The partial mode ( some directories have been desycnrhonized ) tells Jajuk to ignore these ones in the process. Check they are not copied from one device to another.
