General play modes

Before launching tracks, you can set default Jajuk behavior in the command view:

Fig. 01: Modes 1- Repeat mode: if set, selection will be played in a loop 2- Shuffle mode: if set, selection will be played randomly 3- Continue mode: if set, tracks after selection will be played when selection is done 4- Introduction mode: if set, only a part of each track will be played (begin position and play time can bet set in the parameter view)

Note

The " Push " command you will find in most menus allows you to push the selection in the queue: it means the tracks you push will be played when previous selection is over. You can browse the queue using the smart playlist "Queue" (See fig. 02):

Fig.02: Queue

Smart functions

To avoid selecting tracks, you can use these high-level smart functions from command panel or tray:

Play shuffle: launch a shuffle track from the entire collection (only from mounted and ready devices).

Play best of: launch one of your favorite track (learned with dynamic rate) and with a bit of random.

Play novelties: launch one of your newest tracks from your collection with a bit of random.

Continue album: Useful when using shuffle mode and you find an album that you want to play completely.

Selecting tracks to play

  • Select physical or logical perspective (see [pav.html Using perspectives and views])
  • Select one or more tracks to play. There are many ways to select a track:

Using physical or logical tree view

Fig.04: Selecting tracks with tree view

In physical perspective:

  • Select one or more (using control and shift keys) files or directories

In logical perspective:

  • Select one or more (using control and shift keys) genres, artists, albums or tracks

Launching items:

You can:

  • Right click on your selection, a popup menu will appear. Then select 'play' or another command (play repeat, play shuffle, push)
  • Drag and drop selection (only one item) to the queue playlist (see fig. 02)
  • Double-click on a selected item

Using physical or Logical Tracks Table view

Fig. 05: Selecting tracks with table view

In physical perspective:

  • Select one or more files (each row is a file). Use Shift key to perform multiple selections.

In logical perspective:

  • Select one or more tracks (each row is a track). Use Shift key to perform multiple selections.

Launching items:

You can:

  • Right click on your selection, a popup menu will appear. Then select 'play' or another command (play repeat, play shuffle, push, play artist, play album...)
  • Drag and drop selection (only one item) to the queue playlist (see fig. 05)
  • Double-click on a selected item

Using command view

Fig.06: Selecting a track with command view

Using search box

The search box allows you to find any file, track, genre, artists and album... for a given text. A result popup appears. Then launch wanted track clicking on the list.

Using history bar:

The history bar shows all tracks you played (can be cleared and configured in [view_parameter.html the Parameter view]). Click on one track to launch it again.

Controlling player (play, pause, volume...)

Mute: stop/start the sound

Previous Track: Listen previous track in collection. Right click or Shift + left click to play previous album.

Next track: Listen next track in selection. Right click or Shift + left click to play next album.

Fast rewind in current track. Right click or Shift + left click to replay the track

Play / Pause current track

Stop playing. Playing. Queue is cleared

Fast forward in current track

Volume (slider). Note that volume is saved and reset automatically at next Jajuk startup. You can change volume with the mouse wheel over tray popup as well.

Position in current track (slider). Note that current track position is saved and startup track can restart from this position if you check the "Last track at last position" startup mode in Parameter View / startup.

The Jajuk rating system

Jajuk comes with an automatic rating system that learns tracks your like the most based on the frequency you listen to them. Current system works this way: +1 if a track is listen more than 20 seconds, +1 if a track is listen to the end. Notes:

  • Tracks rating is usually displayed as a set of stars in tables (from one to 4 stars). You can access to exact rating value as a tooltip on the stars.
  • You can reset all ratings in the [./view_parameter.html#history Parameter view / History tab].
