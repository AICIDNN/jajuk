Definitions

  • Jajuk is divided into perspectives and views. A perspective is a set of views that can be adjusted at your convenience.
  • Different perspectives have been predefined to make your life easier and will be explained later.
  • To change of current perspective, click on the perspective icon of your choice in the view perspective selector .
  • There are two type of views: static views and contextual views . Contextual views can be added to any perspective using the view menu. Contrary to statics view that can not be changed. Check out the table of contents to get the different views available.

Jajuk perspectives

Jajuk contains following perspectives:

  • Simple perspective : this perspective aims to be the simplest view of Jajuk. On the top you have the view suggestions , and on the bottom the view tracks table .
  • Files perspective : your collection is shown as it is on your disks: files, directories, m3u files, etc. All actions are performed on an given physical file. This perspective is useful when your music don't have tag. But note that if you have several times the same track in different devices (lets say one on your hard disk and one on a CD), you will see it several times in all views. This is the same for playlist files.
  • Tracks perspective : your collection is shown using id3 tags information such as genres, artists, albums, tracks and logical playlists (a logical playlist maps several identical physical m3u playlists). This perspective is only useful if your collection is correctly tagged, otherwise everything will be 'unknown'. In opposition to the Files perspective , you will see tracks instead of files. A track is a logical mapping to one or more files. It means you will see a track only once even if you have several physical files for this track. If you modify a logical playlist, all associated physical playlists (.m3u files) will be modified together.
  • Albums perspective : display albums collection using interactive covers thumbnails.
  • Display perspective : used to show various information while playing tracks: one animation view with large text area to display track names and two cover views.
  • Information perspective : using wikipedia, this perspective gives you information on the current songs, artist and albums.
  • Configuration perspective : this contains views to configure Jajuk (options) and to create or modify devices. A device is a set of physical files (like a directory on your hard disk, a CD, a music player...).
  • Statistics perspective : contains various pies and histograms about your collection: number of tracks, devices sizes, genres repartition, collection size.

Using docking

  • Jajuk interface can be organized just like you want it: you can resize, minimize, drag a view and add it to another view as a tab, make view auto-hideable (using the drawing-pin)...
  • You can reset default views configuration by using the top command view: Views and restore defaults.
  • Views configuration are saved when closing Jajuk.
