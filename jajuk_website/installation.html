Installation guide

Targeted platforms

This documentation targets: Gnu/Linux and others Unix, MS Windows and MacOS.

Requirements

Java Runtime environment (JRE) 1.5 and above. If needed, you can get it for free at http://www.java.com.

Install JRE under Linux

Download the JRE RPM version (.rpm.bin file).
  • Lauch installer : sh j2re-{version}.rpm.bin
  • Log as root and install the RPM : rpm -ivh j2re-{version}.rpm

Download

Download Jajuk last distribution at http://sourceforge.net/project/showfiles.php?group_id=91412.

Support

In case of problem (installing Java JRE for instance), you can emit a support request at http://sourceforge.net/project/stats/?group_id=91412

Installation

o Under Linux

We provide two packages for Linux : Suse RPM (jajuk-${version}-suse.noarch.rpm) and Mandrake RPM (jajuk-${version}-1mdk.noarch.rpm). Suse and Mandrake RPM should work with most others Linux  distributions even if  you may encounter menu icons creation and dependencies issues on JRE with others distributions. To install, use your favorite graphical tool (Yast, Drake...) or log as root and launch rpm -ivh jajuk-${version}.rpm. If system complains for a java2 package and that we already have java installed (check it with command java --version), force installation : rpm -ivh --nodeps jajuk-${version}.rpm

Notes:

  • In both distributions, it can take a while for shortcut to appear in menu.
  • Under Suse, shortcut is created in "Multimedia" menu section.
  • Under Mandrake, shortcut is created in "Multimedia/Sound" menu section.
  • Check your path points to the right JRE (type java --version in case of problem) and not to gcj or others JVM than are not supported by Jajuk, it is a common issue, especially for Fedora, take a look in support database.

o Under Windows

We provide a self-extracting Windows archive (Jajuk-${version}.exe). Double click on it and follow installation.

o Under Mac OS X

  • Note that even after installing java 1.5, Java 1.4.2 is still in use under Mac OS X, you have to display java 1.5 panel control and choose Java 1.5 as the first choice and default JVM. Note that in terminals, 1.4.2 will still be in used even after this change.
  • Use Web Start link : http://jajuk.sourceforge.net/jws.html to install and start Jajuk (Java installer will fail due to JVM 1.4.2 default use).

o Under Linux/Unix, Windows, MacOsX

You can use the provided IzPack Java Installer (jajuk-java-installer-${version}.jar). The distribution file is an executable jar file, the Java version of a 'setup.exe' file but working with any operating system.

  1. Double-click on the downloaded file ( jajuk-java-installer-${version}.jar). If nothing appends, it means .jar extension is not mapped to the java program. Then, you can map it (open with...)  or launch it by typing in a console:  java -jar jajuk-java-installer-${version}.jar
  2. Select your language for installation
  3. Read carefully the provided README file
  4. Read and accept GPL license.
  5. Select installation target : Example: /usr/local/jajuk under Unix,  c:\Program Files\jajuk under Windows.
  6. Create shortcuts (if your OS is supported) : just click on the 'Next' button.

Launching

o Under Linux/Unix

  • (Optional) If you used the IzPack Java installer and that your Unix OS is not supported by IzPack (Linux is supported), you will have to create your own shortcuts if you need them.
  • If you installed using RPM, you should get a shortcut in Applications/Audio under Suse and Multimedia/Sound under Mandrake. You can also launch Jajuk with jajuk (in /usr/bin). It can take a while for shortcuts to appear.
  • Using command line, launch Jajuk with:  sh {jajuk installation path}/jajuk.sh

o Under Windows

Launch Jajuk by clicking on Jajuk icon : Start / Jajuk / Jajuk

Note for MS Windows 95/98/Me users: Some users report problems (error window : "Main-window couldn't be created" ). If this occurs, launch Jajuk by double-clicking on the file {Jajuk installation path}/bin/jajuk.jar. See support request no 938666.

o Under Mac OS X

Use Web Start link.

Upgrade

o Under all systems

If you already have an installed Jajuk release:
  • It is safe to install the new release over previous one without uninstalling, Jajuk supports backwards compatibility. Your collection and configuration is kept and is stored under {your home directory}/.jajuk
  • However Jajuk doesn't support downwards compatibility: A Jajuk release may not work with a collection built by a newer release than the one you use. For downgrading to releases < 0.3.3.4 using exe or jar installer, please uninstall first.

o Under Linux only

To upgrade Jajuk, log as root and launch rpm -Uvh jajuk-${version}-suse.rpm

Uninstallation

o Under Linux/Unix

  • If you used the IzPack Java installer, launch : java -jar {jajuk installation path}/Unistaller/uninstaller.jar
  • If you installed using RPM, use your favorite graphical tool (Yast, Drake...) or log as root and launch: rpm -e jajuk-{version}

o Under Windows

Launch uninstaller by clicking on 'Uninstall' icon next to 'Jajuk ' one

o Under Linux/Unix and Windows

Remove {user home}/.jajuk directory for each user you want to uninstall.
