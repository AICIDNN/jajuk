Jajuk Tester Guide

Last change: 2006/11/05

Welcome to new Jajuk testers ! Have fun.

How many time does it take ?

  • 5 minutes to register
  • About 5 minutes per test case (there are currently from 40 to 50 test cases) : please choose a set of test cases that have not already been tested according the time you have.

How to get support ?

  • Please read the Jajuk user manual
  • In case of doubt, don't hesitate to send your questions at jajuk-support@lists.sourceforge.net

How to become Jajuk tester ?

Jajuk needs testers all the year but especially within the one-month qualification time frame before a major release and few weeks after in order to help finding distribution issues early. Please follow these steps to subscribe to Jajuk Quality Center (to be done only once):
  • Go to http://www.florat.net/testlink/firstLogin.php and create an account in Jajuk Quality Center (this will grant you a read-only access, not enough to perform tests)
  • Send us an e-mail at jajuk-support@lists.sourceforge.net with the platform(s) you plan to use (Windows, Linux or MAC). We will grant you "tester" rights on selected platforms and you will be able to start testing
  • Subscribe to Jajuk list to be aware of new release candidates

How to get last Jajuk release running ?

  • [Linux and Mac only] If under Linux or Mac OS, make sure to get mplayer 1.0 pre8 or later and mozilla 1.7 or later running on your system (nothing needed for Windows).
  • Make sure to have a Java 5 runtime environment (JRE 1.5 or later). If needed, download and install JRE (for free).
  • Launch last Jajuk release available at http://82.239.70.252:8001/jajuk-test/app/ (this link never change, you can bookmark it). This will install and launch Jajuk in Java Web Start mode, next release will be upgraded automatically.
  • Check the Jajuk release in the splashscreen or in the About view (you would continue to use previous release if your client cannot connect to Java Web Start server in a short time frame).

How to test ?

Jajuk quality Center is based on TestLink, a great GPL test case management tool. TestLink manual is available from the "Documentation" tab in the testlink portal after login. Basically, testing is very simple :
  • Log in Jajuk Quality Center
  • Click on "Execute" tab in the TestLink Portal
  • Select test case in the right-side tree by expending categories (choose a "Not tested" test in black: "Passed" tests are in green, "Failed" in red and "Blocked" in blue)
  • Perform the test in Jajuk application
  • Come back to the test page, select result (Passed, Failed, Blocked). If test failed, detail the problem in the text area. Please provide any interesting trace by copy/pasting them from Jajuk (display last traces from Jajuk main menu -> "Help" -> "Show debug traces"). Then save the result using the save button.

Bug tracker system

For Release Candidate (RC) versions (during qualification):

  • We don't use bug tracker in qualification (ie working on release candidate / RC versions), defects are simply noted in the test result field
  • Please do not report RC bugs using SourceForge bug tracker.

For final / production releases

You're welcomed to report issues on production releases using the Jajuk Quality Agent (In Jajuk main menu -> "Help" -> "Report a bug or feature request")

Notes

  • RC releases use the '-test' flag so Jajuk data will be store in <HOME>/.jajuk_test directory and not <HOME>/.jajuk directory used by final releases. This way, you can continue to use stable jajuk and test releases together.
